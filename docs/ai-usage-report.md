# AI Usage Report

## Tools Used and Use Cases
During this assignment, I used multiple AI tools including Claude, ChatGPT, and Gemini to assist with development. These tools helped me extend my portfolio from Assignment 2 into a more advanced application for Assignment 3. AI was used to plan features such as GitHub API integration, project sorting and filtering logic, localStorage state management, and improved form validation. It also helped debug JavaScript issues and refine the CSS design to achieve a cleaner and more modern user interface.

## Benefits and Challenges

### Benefits
AI tools helped speed up development and provided useful suggestions for implementing features such as API integration, filtering, sorting, localStorage, and validation. They also helped identify errors and improve the overall structure and styling of the website.

### Challenges
Some AI-generated code was more complex than necessary or not fully aligned with the assignment requirements. I had to simplify some logic, rename variables more clearly, and reorganize parts of the layout so the code stayed readable and matched my understanding.

### Risks of Using AI
- Over-relying on AI can reduce real understanding of the code
- AI suggestions may include incorrect, inefficient, or unnecessary solutions
- Generated code may not match the exact assignment requirements
- AI-written text can sound too generic if it is not edited

To reduce these risks, I reviewed all generated suggestions, tested every feature manually, simplified code where needed, and only kept parts that I understood and could explain.

## Learning Outcomes
Through using Claude, ChatGPT, and Gemini, I learned more about working with external APIs, handling fetch errors, validating user input, managing state with localStorage, and writing JavaScript with multiple conditions and steps. I also improved my understanding of how to make a portfolio website more dynamic, structured, and user-friendly.

## Responsible Use and Modifications
All AI-generated suggestions were reviewed before being used in the project. I modified the code to fit my own project structure and portfolio content, improved the wording in the documentation, and simplified parts that were too advanced. I also tested all features manually and made sure I understood the final implementation before including it in the project.
